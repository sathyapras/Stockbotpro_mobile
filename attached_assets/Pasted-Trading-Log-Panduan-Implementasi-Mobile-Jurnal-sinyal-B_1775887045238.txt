Trading Log — Panduan Implementasi Mobile
Jurnal sinyal BOW/BOS per device — React Native
KONSEP
Trading Log adalah fitur pencatatan sinyal trading yang diambil user dari platform StockBot Pro. User mencatat entry berdasarkan sinyal BOW atau BOS, kemudian menutup posisi saat sudah exit. Data disimpan di server berdasarkan device_id (UUID unik per device).

Tidak butuh login — identifikasi cukup dengan device_id yang disimpan di expo-secure-store.

SETUP DEVICE ID
import * as SecureStore from "expo-secure-store";
import "react-native-get-random-values";
import { v4 as uuidv4 } from "uuid";
// Panggil sekali saat app pertama kali dibuka
async function getOrCreateDeviceId(): Promise<string> {
  const KEY = "sbp_device_id";
  let id = await SecureStore.getItemAsync(KEY);
  if (!id) {
    id = uuidv4();
    await SecureStore.setItemAsync(KEY, id);
  }
  return id;
}

Simpan device_id di context/store global supaya bisa dipakai semua screen.

BASE URL
const BASE_URL = "https://stockbotpro.replit.app/api";

TIPE DATA
export type SignalType = "BOW" | "BOS";
export type TradeStatus = "OPEN" | "CLOSED";
export interface TradingLogEntry {
  id:          number;
  deviceId:    string;
  ticker:      string;
  signalType:  SignalType;
  entryPrice:  string;   // decimal string, misal "4200.00"
  slPrice:     string;
  tp1Price:    string;
  tp2Price:    string;
  entryDate:   string;   // "YYYY-MM-DD"
  exitPrice:   string | null;
  exitDate:    string | null;
  status:      TradeStatus;
  notes:       string | null;
  createdAt:   string;   // ISO timestamp
}
export interface TradingSummary {
  ok:            boolean;
  totalTrades:   number;
  winRate:       number;  // persen, misal 66.7
  totalPnl:      number;  // kumulatif % P&L dari semua CLOSED trade
  avgHoldDays:   number;
  openPositions: number;
}

2. POST — Tambah trade baru
POST /api/trading-log
Content-Type: application/json

async function addTrade(deviceId: string, trade: {
  ticker:      string;
  signal_type: "BOW" | "BOS";
  entry_price: number;
  sl_price:    number;
  tp1_price:   number;
  tp2_price:   number;
  entry_date:  string;  // "YYYY-MM-DD"
  notes?:      string;
}): Promise<TradingLogEntry> {
  const res = await fetch(`${BASE_URL}/trading-log`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ device_id: deviceId, ...trade }),
  });
  const json = await res.json();
  if (!json.ok) throw new Error(json.error);
  return json.data;
}

Body yang dikirim:

{
  "device_id":   "abc-123",
  "ticker":      "BBRI",
  "signal_type": "BOW",
  "entry_price": 4200,
  "sl_price":    3990,
  "tp1_price":   4600,
  "tp2_price":   5000,
  "entry_date":  "2026-04-01",
  "notes":       "Opsional, catatan bebas"
}

Validasi wajib:

ticker, signal_type ("BOW" atau "BOS"), entry_price, sl_price, tp1_price, tp2_price, entry_date — semuanya wajib
notes — opsional
3. PATCH — Tutup posisi
PATCH /api/trading-log/:id/close?device_id=<uuid>
Content-Type: application/json

async function closeTrade(deviceId: string, tradeId: number, data: {
  exit_price: number;
  exit_date:  string;   // "YYYY-MM-DD"
  notes?:     string;
}): Promise<TradingLogEntry> {
  const res = await fetch(
    `${BASE_URL}/trading-log/${tradeId}/close?device_id=${deviceId}`,
    {
      method: "PATCH",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify(data),
    }
  );
  const json = await res.json();
  if (!json.ok) throw new Error(json.error);
  return json.data;
}

Body yang dikirim:

{
  "exit_price": 4650,
  "exit_date":  "2026-04-10",
  "notes":      "TP1 tercapai, exit sebagian"
}

Setelah berhasil, status trade berubah dari OPEN → CLOSED.

4. DELETE — Hapus trade
DELETE /api/trading-log/:id?device_id=<uuid>

async function deleteTrade(deviceId: string, tradeId: number): Promise<void> {
  const res = await fetch(
    `${BASE_URL}/trading-log/${tradeId}?device_id=${deviceId}`,
    { method: "DELETE" }
  );
  const json = await res.json();
  if (!json.ok) throw new Error(json.error);
}

Response:

{ "ok": true, "message": "Trade berhasil dihapus." }

Hanya bisa hapus trade milik device_id sendiri. Jika ID tidak ditemukan → 404.

KALKULASI P&L DI SISI MOBILE
Untuk tampilan P&L per trade (sebelum ditutup, sebagai preview):

function calcPnlPct(entryPrice: number, currentOrExitPrice: number): number {
  return ((currentOrExitPrice - entryPrice) / entryPrice) * 100;
}
// Warna indikator
function pnlColor(pnl: number): string {
  return pnl > 0 ? "#22c55e" : pnl < 0 ? "#ef4444" : "#94a3b8";
}
